# PkuHoleStudio v0.1.0-alpha.3

这是一次以 Web/TUI 共用后端、真实数据闭环和本地优先体验为重点的预览版本。该版本已经完成 Windows 真实账号、真实归档和真实资料库验收。

## 本版重点

- TUI 与 Web 可通过相同的 `--data-dir` 和 `--db-path` 共用配置、登录会话与 SQLite 资料库。
- Web 登录流程与 TUI 对齐，正确区分 IAAA 和树洞二次短信验证；也可以重新载入 TUI 已保存的本机会话。
- Studio 可原生同步指定 PID 的正文、全部评论和图片，并直接导出带媒体的 archive v2 或 Markdown 阅读包，不依赖 Toolkit。
- archive v2 支持图片校验、本地标签、标签颜色、帖子笔记和评论笔记的导入导出往返。
- 裸数字 PID 和 `#PID` 均可精确搜索；帖子与评论全文搜索支持来源、时间、图片和本地标签筛选。
- 引用关系支持明确 PID、上下文推断、评论引用、前向/反向展示和离线重建。
- 持久任务支持刷新恢复、暂停、继续、取消、失败重试和程序重启后的恢复。
- Web 补齐在线浏览、图片、发洞、回复、点赞、关注、通知、课表、成绩、日志与资料库维护入口。
- AI Provider 支持多个 OpenAI-compatible 配置、运行时切换、连接测试、本地检索问答和课程分析。

完整变更见 [CHANGELOG.md](https://github.com/Susurrium/PkuHoleStudio/blob/v0.1.0-alpha.3/CHANGELOG.md)。

## Windows 安装

1. 下载 `treehole-windows-amd64.zip`。
2. 对照 `SHA256SUMS.txt` 校验压缩包完整性。
3. 解压到一个独立目录，例如 `D:\PkuHoleStudio`。
4. 在解压目录打开 PowerShell，启动 Web：

   ```powershell
   .\treehole-windows-amd64.exe web
   ```

5. 浏览器会打开 `http://127.0.0.1:8080`。默认资料保存在程序工作目录下的 `data` 文件夹。

默认 TUI 启动方式：

```powershell
.\treehole-windows-amd64.exe
```

如果需要让 TUI 与 Web 明确共用资料，请始终为二者传入完全相同的路径：

```powershell
.\treehole-windows-amd64.exe --data-dir D:\PkuHoleStudio\profile --db-path D:\PkuHoleStudio\profile\studio.db
.\treehole-windows-amd64.exe --data-dir D:\PkuHoleStudio\profile --db-path D:\PkuHoleStudio\profile\studio.db web
```

## SHA-256 校验

下载 `SHA256SUMS.txt` 后，在 PowerShell 中执行：

```powershell
(Get-FileHash .\treehole-windows-amd64.zip -Algorithm SHA256).Hash
Get-Content .\SHA256SUMS.txt
```

两处对应的哈希值应相同，字母大小写不影响结果。

## 升级和备份

- 升级前建议退出正在运行的 TUI/Web，并备份原来的 `data` 目录和 SQLite 数据库。
- 新版本可以继续使用现有资料库，数据库迁移会在启动时自动执行。
- 不要把 `cookies.json`、数据库、真实归档、API key、密码或短信验证码发送给他人。

## 说明

- 这是 alpha 预览版，建议保留资料库备份。
- Web 默认仅监听 `127.0.0.1`，不要在不了解风险时改为公网地址。
- AI 默认关闭，只有明确配置 Provider 后才会访问对应模型服务。
- Toolkit 仍可作为独立的简易导出工具，但 Studio 的同步、导入和导出不依赖 Toolkit。
